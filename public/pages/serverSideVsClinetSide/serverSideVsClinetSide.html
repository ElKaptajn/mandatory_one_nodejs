Client-side vs Server-side

Client-side redirection vs. server-side redirection

Client-side redirection is handled by the user's browser and is implemented using JavaScript or HTML. It requires the browser to load the original page before navigating to the new URL. On the other hand, server-side redirection is handled by the Express server, which directly sends an HTTP status code and the new URL to the browser without loading the original page. This results in faster navigation and better SEO.

Fetching in the client vs. fetching in the frontend

Fetching data in the client involves making API calls from the browser, usually using JavaScript. This approach allows for dynamic content updates without a full page refresh, providing a smoother user experience. However the content might not be visible to search engines. Fetching data in the server-side means retrieving data from the server before rendering the HTML. This ensures that the content is available to search engines and visible to users even if JavaScript is disabled, but it may require a full page refresh for content updates.

Structuring HTML with semantic HTML (header, main, footer)

Semantic HTML refers to using appropriate HTML elements to convey meaning and structure to the content. It helps improve the readability and accessibility of a web page. Using semantic elements like <header>, <main>, and <footer> allows developers to define the main sections of a web page, making it easier to understand and maintain. Search engines and screen readers can also better understand the structure of the content, improving SEO and accessibility.
